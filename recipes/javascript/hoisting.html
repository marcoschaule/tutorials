Hoisting

Overview

In JavaScript, variables and functions can be hoisted. Hoisting means, that variable and function declarations are put into memory during the compile phase, but stay exactly where you typed them in your coding. The consequence is that variables and functions exist and can be used before they are declared in the current execution context and all sub contexts.
